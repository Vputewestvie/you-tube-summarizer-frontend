# Настройка Supabase для YouTube Summarizer

## 1. Применение схемы базы данных

Для настройки базы данных выполните следующие шаги:

### Вариант A: Через Supabase Dashboard (рекомендуется)

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard/project/zswoexmjehaaiozvyzeu)
2. Перейдите в раздел **SQL Editor**
3. Нажмите **New query**
4. Скопируйте содержимое файла `supabase-schema.sql` и вставьте в редактор
5. Нажмите **Run** для выполнения SQL-запросов

### Вариант B: Через Supabase CLI

```bash
# Установите Supabase CLI (если не установлен)
npm install -g supabase

# Выполните миграцию
supabase db push --project-id zswoexmjehaaiozvyzeu
```

## 2. Настройка аутентификации

1. В Supabase Dashboard перейдите в **Authentication** → **Providers**
2. Убедитесь, что **Email** provider включен
3. В **Email Auth** настройках:
   - **Confirm email**: Отключите (так как проверки по email не требуется)
   - **Secure email change**: Включите (рекомендуется)

## 3. Проверка таблиц

После применения схемы проверьте, что таблицы созданы:

1. Перейдите в **Table Editor**
2. Должны быть видны таблицы:
   - `users` (с полями: id, email, credits, created_at)
   - `summaries` (с полями: id, user_id, video_url, created_at)

## 4. Тестирование

1. Запустите приложение локально:
   ```bash
   pnpm dev
   ```

2. Откройте http://localhost:3000

3. Протестируйте регистрацию:
   - Нажмите "Зарегистрироваться"
   - Введите email и пароль (минимум 6 символов)
   - После регистрации должен создаться профиль с 5 кредитами

4. Протестируйте вход:
   - Войдите с созданными учетными данными
   - В шапке должно отображаться количество кредитов

5. Протестируйте создание summary:
   - Вставьте ссылку на YouTube видео
   - Нажмите "Сократить"
   - Кредиты должны уменьшиться на 1

## 5. Функции базы данных

Схема создает следующие функции:

- `handle_new_user()` - автоматически создает профиль пользователя с 5 кредитами при регистрации
- `decrement_credits(user_id)` - уменьшает количество кредитов на 1

## 6. Безопасность

Включен Row Level Security (RLS):
- Пользователи могут видеть и редактировать только свои данные
- Пользователи могут видеть только свои summaries
- Все операции проверяют права доступа

## Troubleshooting

### Ошибка "Недостаточно кредитов"
- Проверьте, что триггер `on_auth_user_created` создан и работает
- Убедитесь, что при регистрации создается запись в таблице `users`

### Ошибка аутентификации
- Проверьте, что `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` установлены в `.env.local`
- Убедитесь, что Email provider включен в Supabase Dashboard

### Ошибка при создании summary
- Проверьте, что пользователь авторизован
- Убедитесь, что у пользователя есть кредиты (> 0)
